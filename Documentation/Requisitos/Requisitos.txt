REQUISITOS:
	FUNCIONAIS
		RQ_FUNC01 :: Gerar um Formulário de empréstimo (em PDF, após o empréstimo estar concluído)
			Descrição: Um formulário como comprovante de empréstimo, gerado em tempo real pelo programa, onde deverá conter:
				- data de empréstimo;
				- data de devolução;
				- dados do emprestante;
				- Dados do objeto em empréstimo:
					-Quantidade, Nº do Patrimonio, Descrição, Localização,
					 Nº da Nota Fiscal, Data de Aquisição, Fornecedor,
					 Valor R$, Grau de Depreciação.
				- assinatura do responsável;
				- dados da instituição.
			Entrada: Os dados inseridos estarão na tabela de empréstimos, os valores serão dispostos conforme o modelo do formulário e de acordo com os dados inseridos no momento do empréstimo.
			Origem: A partir da tela de finalização de cadastro e na tabela de empréstimos.
			Saídas:	Um arquivo PDF com todos os dados do empréstimo (data de empréstimo e devolução), dados do emprestante, dados da instituição e dados do item emprestado. 
			Destino: Para caso o emprestante quiser manter um comprovante de seu empréstimo em mãos.
			Ação:   Poderá ser solicitado logo após a finalização ou ao acessar as tabelas de empréstimos realizados, com uma opção de gerar formulário.
			Requer: Solicitação do usuário, então o sistema ira criar o formulário de imediato, inserindo seus valores.

		REQ_FUNC02 :: Cadastro do tipo de usuário:
			Descrição: Criação de duas classes distintas com o objetivo de utilização do sistema em diferentes níveis de acesso e autonomia, sendo eles:
			- Admin:
				- Controle e integração de todos os dados;
				- Todos os Funcionários;
				- criar conta de funcionário.				
			- Funcionário:
				- cadastro de emprestante;
				- empréstimo;
				- inserção de item.
			Entrada: ID distinta(por categoria), login e senha para acesso ao sistema.
			Origem: Acesso a todas as telas de cadastros e planilhas.
			Saída: Contas de usuários com login e senha em diferentes níveis de acesso.
			
		REQ_FUNC03 :: Arquivo de log
			Descrição: Registros de todas as atividades do usuário
			Origem: Ações realizadas pelos usuarios do sistema
			Saídas: Arquivo detalhando as atividades 
			Ação: Gera um arquivo para verificar as atividades realizadas por cada usuário dentro do sistema (data, horário, descrição, nome e registro de usuário, respectivamente).


		REQ_FUNC04 :: Níveis de acesso
			-FUNCIONÁRIO
				- FUNCIONÁRIO Cadastro de emprestante:
					- Nome, CPF, telefone, e-mail, endereços (rua, cidade, estado, CEP).
				- FUNCIONÁRIO Cadastro de item:
					- Quantidade, Nº do Patrimonio, Descrição, Localização,
					  Nº da Nota Fiscal, Data de Aquisição, Fornecedor,
				  	  Valor R$, Grau de Depreciação.
			-ADMIN	
				- ADMIN Cadastro de Funcionário:
					- Nome, CPF, data de nascimento, telefone, e-mail, endereços (rua, cidade, estado, CEP);
					- Senha.
				- ADMIN Cadastro de Administrador:
					- Nome, CPF, data de nascimento, telefone, e-mail, endereços (rua, cidade, estado, CEP);
					- Senha.
				- Banco de Dados:
					- Cópia do arquivo .db;
					- Arquivo principal do Banco de Dados.
					
		REQ_FUNC05 :: Opção de abrir um arquivo de planilha (.xls e .csv serão suportados).
				Descrição: O sistema fara uma leitura dos dados na planilha e inserí-los no Banco de Dados, lidando com dados incosistentes e duplicados e gerenciando as quantidades dos itens, mantendo uma organização e separação de dados individuais (nota Fiscal, Fornecedor, Nº do Patrimonio, data de aquisição etc.). Tal planilha será armazenada em cópia dentro da pasta "tmp", por motivos de backup e evitar a perda de dados no arquivo original;
				Entradas: Arquivos de planilhas com extensão xls(Excel) ou csv(separado por vírgula ou ponto e vírgula);
				Origem: Arquivos já criados no sistema;
				Saída: A pls
			REQ_DATABASE02 :: Métodos de Acesso.
				Descrição: Funções de Adição, Remoção, Edição e Visualização flexíveis.
				Origem: Implementação na Linguagem.
			
			COM RELAÇÃO AO BANCO DE DADOS:
			REQ_DATABASE01 :: Quatro Tabelas. (Clientes, Usuários, Empréstimos, Itens)
				Descrição: Criação de um Banco de Dados formado por quatro tabelas, que se relacionem umas com as outras
				Entradas: Dados de Itens, Clientes, Empréstimos e Usuários
				Origem: Dados pré-cadastrados e dados fornecidos pelo cliente
				Saídas: Tabelas
				
			REQ_DATABASE02 :: Métodos de Acesso
				Descrição: Funções de Adição, Remoção, Edição e Visualização flexíveis
				Origem: Implementação na Linguagem

		 COM RELAÇÃO A INTERFACE:
			REQ_INTERFACE01 :: Visualização das tabelas do banco de dados;
				Descrição: O sistema deve mostrar as tabelas geradas pelo Banco, sendo atualizadas conforme o usuário a modifica.
				Entradas: O usuário deverá abrir a aba da tabela desejada. O sistema deverá mostrar todos os dados da tabela dentro do Banco de Dados.
				Origem: Dados da tabela desejada, dentro do Banco.
				Saída: Visualização da tabela completa.
				Requer: Acionamento de um botão para abrir a aba da tabela.
			
			REQ_INTERFACE02 :: Opção de abrir planilha;
				Descrição: O usuário de ter o botão para abrir um arquivo de plailha para ser editado no sistema.
				Entradas: O usuário irá abrir a tabela desejada, que ficará salva com uma cópia.
				Origem: Deverá ter no próprio computador.
				Saídas: Planilha atualizada.


			REQ_INTERFACE03 :: Opção de Salvamento, onde o sistema irá salvar os dados no Banco, mantendo a planilha aberta inalterada;
				Descrição: Para que as alterações dentro da planilha sejam efetivadas, o usuário precisa acionar o botão salvar.
				Destino: O sistema irá atualizar as planilhas abertas.


			REQ_INTERFACE05 :: Opção de Gerar um Formulário em PDF (podendo ser impresso)
				Descrição: Essa opção deve ser mostrada ao realizar um empréstimo e ao acessar o Banco de Dados de Empréstimos, deverá ter a opção de gerar um formulário.
				
			COM RELAÇÃO AS TELAS
				REQ_TELAS01 :: Tela de Login;
					Descrição: Tela onde o usuário fará o login e acessará o sistema.
					Entradas: Nome de usuário e senha.
					Origem: Ficará salvo no Banco de Dados de Usuário.
					Saídas: O sistema irá salvar a atividade no arquivo de log e o usuário poderá acessar o sistema.
					Destino: Necessário para poder acessar o sistema.
					Requer: O usuário precisa estar cadastrado no sistema e suas informações devem bater com as que o usuário está inserindo.
				
				REQ_TELAS02 :: Tela Inicial, Home;
					Descrição: Tela onde o usuário fará todas as ações, visualizar planilhas, abrir outras página etc.
					Requer: O usuário precisa estar logado no sistema para acessá-la.

				REQ_TELAS03 :: Tela de Cadastros;
					Descrição: Tela onde o usuário fará o cadastro de itens, clientes ou empréstimos. Terá opções de qual tipo de dado inserir, assim o sistema mostrará os campos necessários.
					Entradas: Informações a serem inseridas nos campos.
					Saídas: Os dados serão inseridos dentro do Banco de Dados.
					Requer: O usuário precisa estar logado.


	NÃO FUNCIONAIS
		- Mensagens de erro devem ser escritas de forma clara, para que o usuário entenda perfeitamente onde ocorreu o erro e a possível solução.
		- Segurança: 
			- As senhas devem ser armazenadas em formato criptografado dentro do Banco de Dados;
		- Telas devem ser limpas e claras, para facilitar o uso.
